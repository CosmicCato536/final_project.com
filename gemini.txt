Работа с таблицами «многие ко многим» отличается от обычной тем, что теперь для выполнения одного действия (например, создания персонажа с мечом) нужно сделать два шага.
Вот основные сценарии работы с примерами кода:
## 1. Добавление данных (INSERT)
Сначала мы создаем сам объект (персонажа, оружие или способность), а затем создаем запись-связку в промежуточной таблице.

import sqlite3
conn = sqlite3.connect("data.db")cursor = conn.cursor()
# Шаг 1: Создаем персонажа и получаем его ID
cursor.execute("INSERT INTO character (name, hp) VALUES (?, ?)", ("Арагорн", 100))char_id = cursor.lastrowid # Получаем ID только что созданного героя (например, 1)
# Шаг 2: Допустим, в базе уже есть способности с ID 5 и 7# Создаем связи в промежуточной таблицеabilities_to_add = [5, 7]for ab_id in abilities_to_add:
    cursor.execute("INSERT INTO character_abilities (character_id, ability_id) VALUES (?, ?)", (char_id, ab_id))

conn.commit()

------------------------------
## 2. Получение данных (SELECT + JOIN)
Чтобы вытащить персонажа вместе с его способностями, нужно «склеить» три таблицы через JOIN.
Запрос: «Покажи имя персонажа и названия всех его способностей»

SELECT character.name, abilities.textFROM characterJOIN character_abilities ON character.id = character_abilities.character_idJOIN abilities ON character_abilities.ability_id = abilities.idWHERE character.id = 1;

------------------------------
## 3. Обновление данных (UPDATE / DELETE)
В этой схеме вам редко нужно что-то менять в самих связях. Если персонаж потерял способность, вы просто удаляете строку из таблицы-связки:

# Забрать у персонажа (ID: 1) способность (ID: 5)
cursor.execute("DELETE FROM character_abilities WHERE character_id = ? AND ability_id = ?", (1, 5))

Если вы хотите заменить все старые способности на новые:

   1. Удаляете все строки из character_abilities для этого character_id.
   2. Вставляете новые строки с актуальными способностями.

------------------------------
## Главные советы по работе:

* Транзакции: Всегда делайте conn.commit() только после того, как выполнили оба действия (создали персонажа И создали связи). Если первое сработает, а второе — нет, данные будут неполными.
* Удаление персонажа: Благодаря ON DELETE CASCADE (который мы прописали в коде ранее), если вы удалите строку из таблицы character, все его связи в character_abilities и character_weapons исчезнут автоматически. Вам не нужно чистить их вручную.
* DB Browser: Чтобы проверить, всё ли работает, просто откройте таблицу character_abilities. Если там есть строки, где один и тот же character_id встречается с разными ability_id — поздравляю, связь «многие ко многим» работает!
